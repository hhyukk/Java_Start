# 변수 시작
#### 패키지란?
자바 파일을 **폴더 단위로 구분**하는 개념.

예:`package variable;` → `variable` 폴더 안에 위치한 클래스임을 명시.

#### 변수 없이 숫자 출력
```java
System.out.println(10);
System.out.println(10);
System.out.println(10);

```
숫자 10을 3번 출력.

숫자를 20으로 바꾸려면 모든 출력문을 수정해야 함.

#### 변수 도입 이유
**동일한 값을 여러 번 사용할 때** 유지보수가 쉬움.

**값을 한 번만 수정**해도 전체 코드에 반영됨.

사용자의 입력처럼 **변하는 값**을 저장할 때 필수.

```java
int a;     // 변수 선언 (정수형 변수 a 생성)
a = 10;    // 변수 초기화 (a에 10 저장)

System.out.println(a); // 10 출력
System.out.println(a); // 10 출력
System.out.println(a); // 10 출력
```
`int` → 정수(int형)를 저장하는 변수 타입.

`a = 10 `→ 변수 a에 숫자 10 저장 (=은 '저장'의 의미).

`System.out.println(a)` → 변수에 저장된 값(10)을 읽어 출력.

#### 변수의 장점
값이 변경되어도 변수만 수정하면 됨 → 유지보수 용이

반복 사용 가능

사용자의 입력 등 **변동성 있는 값**을 다루기 쉬움

# 변수 값 변경
#### 변수는 "변할 수 있는 값"을 저장하는 공간
변수에 저장된 값은 **언제든지 변경 가능**함.

변수 값을 바꾸면 **이전 값은 삭제**되고 새로운 값으로 대체됨.

![](https://velog.velcdn.com/images/hhyukk/post/0f6a6c71-f0d8-4e02-b25c-5167678261af/image.png)


```java
package variable;
public class Var3 {
    public static void main(String[] args) {
        int a;     // 변수 선언
        a = 10;    // 변수 초기화 (a = 10)
        System.out.println(a);  // 10 출력

        a = 50;    // 변수 값 변경 (a = 50)
        System.out.println(a);  // 50 출력
    }
}
```
#### 실행 결과
```
10
50
```
`a = 10;` → 변수 `a`에 10 저장

`System.out.println(a);` → 변수 `a`의 값(10)을 출력

`a = 50;` → 변수 `a`의 값을 50으로 변경 (**기존 값 10은 사라짐**)

`System.out.println(a);` → 변수 `a`의 값(50)을 출력

# 변수 선언과 초기화
#### 변수 선언이란?
변수를 선언하면 **컴퓨터 메모리에 공간**을 만들고, 해당 공간에 **이름을 부여**함.

이후 이 **이름(변수명)**을 통해 값을 저장하거나 읽을 수 있음.

```java
int a;     // 정수형 변수 a 선언
int b;     // 정수형 변수 b 선언
int c, d;  // 한 줄에 여러 변수 선언
```

#### 변수 초기화란?
**초기화**: 선언한 변수에 **처음으로 값을 저장**하는 것

```java
int a;     // 선언
a = 1;     // 초기화

int b = 2; // 선언과 동시에 초기화

int c = 3, d = 4; // 여러 변수 동시 선언 및 초기화
```

#### 변수 초기화는 필수!
**초기화하지 않은 변수는 사용할 수 없음.**

컴파일 에러 발생:
```java
int a;
System.out.println(a); // 컴파일 에러 발생!
```

자바에서는 변수 사용 전에 **초기화가 강제됨**

이유: **초기화되지 않은 메모리 공간은 예측 불가한 값**을 가질 수 있어서 안전하지 않음

### 참고 용어
| 용어                         | 설명                              |
| -------------------------- | ------------------------------- |
| **변수 선언**                  | 메모리 공간을 만들고 이름을 부여              |
| **초기화**                    | 변수에 값을 처음 저장하는 것                |
| **컴파일 에러**                 | 자바 문법 위반 시 발생하는 오류              |
| **지역 변수 (Local Variable)** | 메서드 내부에서 선언된 변수. 반드시 직접 초기화해야 함 |

### 요약
- 변수는 **선언 후 초기화**해야 사용 가능
- **여러 변수**를 한 줄에 선언 및 초기화 가능
- 초기화하지 않고 사용 시 **컴파일 에러 발생**
- 지역 변수는 자바가 **자동 초기화하지 않음**
# 변수 타입 1
#### 변수 타입(Type)이란?
변수는 **저장할 값의 종류(형식)**에 따라 타입이 정해짐.

타입은 **정수, 실수, 문자, 논리, 문자열** 등을 다룰 수 있도록 구분되어 있음.

| 타입        | 설명    | 예시                | 사용법                   |
| --------- | ----- | ----------------- | --------------------- |
| `int`     | 정수형   | `1`, `100`        | `int a = 100;`        |
| `double`  | 실수형   | `0.5`, `10.75`    | `double b = 10.5;`    |
| `boolean` | 논리형   | `true`, `false`   | `boolean c = true;`   |
| `char`    | 문자 1개 | `'A'`, `'가'`      | `char d = 'A';`       |
| `String`  | 문자열   | `"Hello"`, `"자바"` | `String e = "Hello";` |

> String은 대문자로 시작하는 클래스 타입이며, 문자열을 다루기 위한 특별한 타입입니다.

```java
int a = 100;
double b = 10.5;
boolean c = true;
char d = 'A';
String e = "Hello Java";

System.out.println(a); // 100
System.out.println(b); // 10.5
System.out.println(c); // true
System.out.println(d); // A
System.out.println(e); // Hello Java
```

#### 타입에 맞는 값만 저장해야 함
```
int a = "백원";   // X: 문자열은 int 타입에 저장 불가
int b = "100";    // X: 문자열 "100"은 숫자가 아님
int a = 100;      // O
```
### 리터럴 (Literal)이란?
코드에서 **직접 적은 고정된 값.**

| 리터럴 종류  | 예시              |
| ------- | --------------- |
| 정수 리터럴  | `100`           |
| 실수 리터럴  | `10.5`          |
| 불리언 리터럴 | `true`, `false` |
| 문자 리터럴  | `'A'`           |
| 문자열 리터럴 | `"Hello Java"`  |

> 변수의 값은 변할 수 있지만, **리터럴 자체는 고정된 값**이다.
### 요약
- 변수는 **타입에 맞는 값만 저장** 가능
- `String`은 문자열을 위한 **특별한 타입**
- 코드 안에 적힌 고정된 값은 **리터럴**
- 잘못된 타입의 값을 저장하면 **컴파일 오류 발생**

# 변수 타입 2
#### 숫자 타입의 종류와 메모리 크기
- 정수형 (음수, 0, 양수)

| 타입      | 범위                                      | 메모리   | 비고                              |
| ------- | --------------------------------------- | ----- | ------------------------------- |
| `byte`  | -128 \~ 127                             | 1byte | 거의 사용하지 않음. 파일 처리 시 사용          |
| `short` | -32,768 \~ 32,767                       | 2byte | 거의 사용하지 않음                      |
| `int`   | -2,147,483,648 \~ 2,147,483,647 (약 20억) | 4byte | **자바에서 기본 정수 타입**               |
| `long`  | ±9,223,372,036,854,775,807              | 8byte | **20억 이상 정수를 다룰 때 사용** (`L` 필수) |

- 실수형 (소수점 있는 수)

| 타입       | 범위 (대략)  | 정밀도  | 메모리   | 비고                     |
| -------- | -------- | ---- | ----- | ---------------------- |
| `float`  | ±3.4E38  | 7자리  | 4byte | `f` 필요, 거의 사용하지 않음     |
| `double` | ±1.7E308 | 15자리 | 8byte | **실수는 기본으로 double 사용** |

- 기타 타입

| 타입        | 설명                                     |
| --------- | -------------------------------------- |
| `boolean` | `true`, `false`만 저장 가능. 조건문에서 자주 사용    |
| `char`    | 문자 1개 저장 (`'A'`, `'가'` 등). 거의 사용 안 함   |
| `String`  | 문자열 저장. **문자 하나든 여러 개든 보통은 String 사용** |

#### 리터럴 타입 지정

| 타입       | 기본 리터럴 형태 | 형 지정 필요 여부     | 예시                       |
| -------- | --------- | -------------- | ------------------------ |
| `int`    | 100       | × (기본이 int)    | `int a = 100;`           |
| `long`   | 큰 숫자      | `L` 필수         | `long l = 12345678900L;` |
| `double` | 10.0      | × (기본이 double) | `double d = 10.0;`       |
| `float`  | 10.0      | `f` 필수         | `float f = 10.0f;`       |

> **소문자 `l`은 숫자 `1`과 혼동되므로 `L` 사용 권장**

#### 잘 안 쓰는 타입

| 타입              | 이유                         |
| --------------- | -------------------------- |
| `byte`, `short` | 표현 범위 작음, 거의 사용 안 함        |
| `float`         | 정밀도 낮음, `double` 사용 추천     |
| `char`          | 문자 1개만 저장, 대부분 `String` 사용 |

### 요약
- 숫자 타입은 표현 범위에 따라 여러 가지가 있음.
- 정수는 **int**, 실수는 **double**을 기본으로 사용.
- 큰 정수는 **long + L**, float은 **f**를 붙여야 함.
- 실제 개발에서는 **int, long, double, boolean, String**만 주로 사용.
- 나머지 타입은 거의 사용하지 않으며, `byte`는 파일 다룰 때 예외적으로 사용됨.

# 자바 변수 이름 짓기 – 규칙과 관례

#### 변수 명명 규칙 (필수, 지키지 않으면 오류 발생)
| 규칙 내용         | 예시                                                  |
| ------------- | --------------------------------------------------- |
| 숫자로 시작할 수 없음  | (X) `1num`, `7age` <br>(O) `num1`, `age7`               |
| 공백 사용 불가      | (X) `user name`                                       |
| **예약어 사용 금지** | (X) `int`, `class`, `public` 등                        |
| 사용 가능한 문자     | **영문자(a-z, A-Z), 숫자(0-9), 밑줄(\_)**, **달러(\$)** 만 가능 |

#### 변수 명명 관례 (전 세계 개발자가 따름 → 사실상 규칙)
| 항목                    | 내용                                             | 예시                                         |
| --------------------- | ---------------------------------------------- | ------------------------------------------ |
| 소문자로 시작               | 변수 이름은 소문자로 시작                                 | (O) `count`, `userName`                      |
| **낙타표기법(Camel Case)** | 여러 단어를 붙일 때, **첫 단어는 소문자**, 나머지 단어는 **대문자 시작** | (O) `orderDetail`, `studentCount`            |
| **의미 있는 이름 사용**       | 변수의 목적이 드러나야 함                                 | (O) `maxScore`, `userAge`<br>(X) `a`, `b`, `c` |

#### 낙타표기법(Camel Case)이란?
여러 단어를 **띄어쓰기 없이 연결**하면서, 두 번째 단어부터는 **첫 글자를 대문자**로 작성

낙타의 등처럼 올라간다고 해서 붙여진 이름

> Ex) studentName, userAccount, orderDetail

#### 자바 명명 규칙 요약
| 항목             | 규칙                                                   |
| -------------- | ---------------------------------------------------- |
| **클래스 이름**     | **첫 글자 대문자** + 낙타표기법 → `Person`, `OrderDetail`       |
| **변수, 메서드 이름** | **첫 글자 소문자** + 낙타표기법 → `userAccount`, `maxScore`     |
| **상수**         | **전부 대문자 + 언더바(\_)로 구분** → `MAX_SPEED`, `USER_LIMIT` |
| **패키지 이름**     | **전부 소문자** → `com.example.app`                       |

### 요약
- **규칙**: 숫자 시작 금지, 공백·예약어 금지, 유효한 문자만 사용
- **관례**: 낙타표기법 사용, 소문자 시작, 의미 있는 이름 부여
- 클래스는 **대문자 시작**, 변수·함수는 **소문자 시작**
- **상수는 전부 대문자**, 패키지는 **소문자**
